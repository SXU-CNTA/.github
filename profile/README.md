# 计算机与网络技术协会 · GitHub 社区

本 GitHub 组织是 **[计算机与网络技术协会]** 的官方技术开源社区。

## 🎯 我们的目标

- 沉淀协会的技术资产与竞赛成果
- 提供一个安全、规范的协作环境
- 鼓励 **自愿、高质量** 的技术分享

## 👥 成员角色与分工
| 角色 | GitHub 对应身份 | 核心职责 | 权限说明 |
| :--- | :--- | :--- | :--- |
| **协会负责人** | `Security Manager` | 组织层面的安全管理，清理违规/废弃仓库 | • 可删除任意仓库<br>• **不可**修改他人仓库代码<br>• **不可**分配成员权限 |
| **项目负责人** | 仓库 `Admin` | 负责具体项目的代码审核、分支保护、合并 PR | • 拥有所负责仓库的完整管理权<br>• 可审核/合并 PR<br>• 可设置该仓库的分支保护规则 |
| **协会成员** | `Member` + 仓库 `Write` | 参与项目开发，提交代码 | • 可 push 分支、发起 PR<br>• 不可合并 PR<br>• 不可修改仓库设置 |
| **校外访客** | `Outside collaborator` + `Read` | 只读访问 | • 仅可查看代码，不可提交 |
## 🔐 加入要求

**必须满足「会使用 GitHub」**，包括但不限于：
- Git 基础操作（clone / branch / commit / push）
- Fork + Pull Request 流程
- 理解 PR 审核与冲突处理

> 不满足门槛者，暂不允许加入组织。可通过协会培训后再申请。

## 🧩 统一贡献流程（建议）

1. **Fork** 目标仓库到个人账号
2. **Clone** 自己 fork 的仓库到本地
3. 创建独立分支（命名规范见下）
4. 提交清晰、原子化的 commit（规范见下）
5. **Push** 到个人远程仓库
6. 在 GitHub 网页端发起 **Pull Request** 到协会主仓库的 `main` 分支
7. 等待仓库所有者审核
### 注：具体仓库怎么管理以仓库所有者的安排为准

> ❌ 严禁直接 push 到 `main` 主分支

### 分支命名规范（建议）

- `feature/xxx`：新功能开发
- `fix/xxx`：修复 bug
- `docs/xxx`：仅文档更新

### Commit 信息规范（建议）

- `feat: 添加用户登录接口`
- `fix: 修复表格渲染错误`
- `docs: 更新 README 贡献流程`

## ⚖️ 责任说明

- **自己的代码自己负责**：若因个人改动破坏代码，由贡献者承担修复责任
- 仓库所有者拥有 `main` 分支的完全合并权限
- 组织不承担因代码错误导致的竞赛或项目损失
- 禁止提交含有敏感信息（密码、API 密钥）的代码

## 📌 社区规定

1. 所有 PR 必须描述清晰，不得提交无关修改（建议）
2. 禁止恶意破坏、覆盖他人内容
3. 校外访客需通过微信群 / QQ群 / 邮件联系负责人申请，仅授予只读权限
